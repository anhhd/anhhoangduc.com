33  MLOPs với vetiver

Khi làm việc với MLOPS ta sẽ cần lưu trữ metadata, nhiều version khác nhau của features, tập dữ liệu đầu vào cũng như cần phải deploy mô hình dưới dạng API hoặc docker. Với sự kết hợp của pins và vetiver sẽ cho phép đơn giản hóa toàn bộ ML flow cho production.

  • pins cho phép lưu trữ và đánh dấu các phiên bản của các đối tượng khác nhau, bao gồm cả dữ liệu cũng như mô hình
  • vetiver cho phép đơn giản hóa toàn bộ flow của MLOPs

pins và vetiver là 2 packages của RStudio giúp chuẩn hóa flow tạo MLOPs trên cả R & Python.

MLOPS lưu trữ các phiên bản của model artifacts: metadata, model, code, data

33.1 Lưu trữ object với pins

Ý tưởng của pins rất đơn giản, pins sẽ có 3 nhóm câu lệnh đơn giản:

  • board_XXX: Tạo một board, là nơi chưa các object sẽ được lưu trữ. Trong các tổ chức sử dụng mạng LAN là nơi chia sẻ dữ liệu, có thể dùng board_folder.
  • pin_write: Cho phép write/save một object vào board để có thể chia sẻ
  • pin_read: Cho phép đọc một object từ một board nhất định.

Xem ví dụ về workflow dưới đây giữa 2 người phân tích A & B

Với người A

library(pins)
# Step 0: Tạo board - cho phép lưu trữ version các object
board <- board_folder("./board", version = T)

# Step 1: Lưu object mtcars - version 1
board %>% 
  pin_write(head(mtcars), "mtcars")

# Step 2: Lưu object mtcars - version 2
board %>% 
  pin_write(tail(mtcars), "mtcars")

# Tìm kiếm version của mtcars
board %>% 
  pin_versions("mtcars")

Với người B:

library(pins)
# Step 0: Tạo kết nối đến board đã có sẵn
board <- board_folder("./board", version = T)

# Step 1: Đọc object mtcars đã có sẵn
board %>% 
  pin_read("mtcars", version = "xxx")

33.2 MLOPS với vetiver

Với MLOPS, vetiver sẽ cho phép thực hiện:

  • Lưu trữ & versioning mô hình
  • Deploy model dạng API từ pin board
  • Khởi tạo docker file phục vụ deploy API trên môi trường production với docker

33.3 Tạo mô hình đơn giản

Xây dựng & lưu trữ 2 mô hình mtcars

library(tidymodels)
library(vetiver)
library(tidyverse)
library(pins)

# Step 0: Tạo board (tạo board_temp cho notebook - thực tế cần tạo board_folder)
model_board <- board_temp(versioned = TRUE)

# Step 1: Tạo & lưu trữ mô hình LM
car_mod <- workflow(mpg ~ ., linear_reg()) %>%
  fit(mtcars)

v <- vetiver_model(car_mod, "cars_mpg")

model_board %>% vetiver_pin_write(v)

# Các meta data có thể lấy ra từ list của v

# Step 2: Xây dựng mô hình mới với vetiver
car_mod <- workflow(mpg ~ ., decision_tree(mode = "regression")) %>%
  fit(mtcars)

v <- vetiver_model(car_mod, "cars_mpg")

model_board %>% vetiver_pin_write(v)

# Step 3: Kiểm tra các version
model_board %>% pin_versions("cars_mpg")
  • Tạo ra API
library(plumber)
v <- model_board %>% vetiver_pin_read("cars_mpg")

# Model API từ plumber route
pr() %>%
  vetiver_api(v)
  • Run API với port 8080
pr() %>%
  vetiver_api(v) %>% 
  pr_run(port = 8080)

33.4 Tạo docker file

vetiver hỗ trợ tạo nhanh docker file, bao gồm 3 cấu phần sau:

  • vetiver_renv.lock: Chứa các dependency của môi trường R
  • plumber.R: Script đọc và serve model từ pin board
  • Dockerfile: File docker, cho phép đọc các dependency và tạo ra Docker Image với Docker Compose
vetiver_prepare_docker(model_board, "cars_mpg")
