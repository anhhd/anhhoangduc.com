55  Spatial data

55.1 Giới thiệu

Spatial data là loại dữ liệu chứa các thông tin về “thuộc tính hình học”(geometry), “hình dạng” (shape) hoặc “vị trí”(location) của một đối tượng địa lý và thường kết hợp với các attributes chứa thông tin liên quan khác. Định dạng dữ liệu spatial data cho phép chúng ta xây dựng các biểu đồ với R hiệu quả.

Spatial Analytics có rất nhiều cách thức phát triển khác nhau. Tuy nhiên, để thuận tiện và dễ dàng trong việc ứng dụng, ta tập trung vào 2 nhóm sau.

  • Biểu đồ tĩnh (static map): sf, tmap, ggplot2
  • Biểu đồ động (interactive map): leaflet

55.2 Cấu trúc dữ liệu spatial

Một loại định dạng dữ liệu được dùng phổ biến với spatial data là vector data. Loại dữ liệu này thường được lưu dưới dạng shape (.shp),để đọc định dạng này chúng ta sử dụng function read_sp trong package sf

library(sf)
library(tidyverse)
lnd_sport <- sf::read_sf("data/spatial-data/london_sport.shp")
print(lnd_sport)

Định dạng dữ liệu này bao gồm hai cấu phần chính:

  • geometry: Là tật hợp các điểm kinh độ và vĩ độ xác định đường biên của một đối tượng trên hệ trục tọa độ.
  • attribute: Các dữ liệu thông thường của đối tượng (quan sát) được chứa trong raster.

55.3 Xây dựng biểu đồ tĩnh

55.3.1 Biểu đồ cơ bản

tmap là package phổ biến nhất để vẽ các biểu đồ tĩnh, tương tự như ggplot2, package tmap cũng xây dựng theo hướng tạo các lớp chồng lấn.

library(tmap)
data(NLD_muni, NLD_prov)

tm_shape(NLD_muni) + 
#fill by population per km2
    tm_fill("population",
           convert2density=TRUE,
           style="kmeans",
           palette = "Greens",
           title="Population per km2") + 
  tm_borders(alpha= .5) +  
#combine with province data 
tm_shape(NLD_prov) +
  tm_borders(lwd=2, alpha = .5) +
  tm_text("name", 
          size = .6, 
          alpha = 0.9, 
          shadow=TRUE,
          bg.color="white", 
          bg.alpha=.25) +
  tm_layout(asp = 0)

55.3.3 Vẽ biểu đồ với ggplot2

Trước khi có package sf, khi cần vẽ map bằng ggplot2 ta cần convert dữ liệu spatial về dạng data.frame, tuy nhiên version mới của ggplot2 đã cập nhật thêm thuộc tính geom_sf - có thể vẽ trực tiếp trên sf objects mà không cần convert về data.frame.

library(tmap)
data(NLD_muni,NLD_prov)
library(dplyr)
library(scales)
library(RColorBrewer)

pal <- brewer.pal(4, "YlGn")

NLD_muni %>% 
  st_as_sf() %>% # Convert sang sf
  ggplot() +
  geom_sf(aes(fill = population),
          color = "light grey") +
  scale_fill_gradientn(colors = pal, trans = "log10",
                         name ="population per km2", labels = comma) +
  theme_minimal() +
  labs(title = "Population density in London")

55.4 Xây dựng biểu đồ động với leaflet

leaflet là package cho phép chúng ta xây dựng biểu đồ động (interactive) rất chuyên nghiệp với dữ liệu đầu vào là tọa độ địa lý theo từng điểm. leaflet sử dụng pipe operator trong việc tạo các lớp biểu đồ

library(dplyr)
library(leaflet)
load("data/location_data.rda")
load("data/branch_address.rda")
leaflet() %>% 
  addProviderTiles(providers$CartoDB.Positron) %>%
  setView(108.2772, 14.05832, zoom = 5) %>% 
  addMarkers(data = branch_address,
             lng = ~lon_branch, 
             lat = ~lat_branch,
             popup = ~as.character(branch_name)) %>% 
   addHeatmap(data = location_data, 
             lng = ~lon, 
             lat = ~lat, 
             radius = 7, 
             blur = 22) %>% 
  addCircleMarkers(data = location_data,
                   lng = ~lon, lat = ~lat, 
                   clusterOptions = markerClusterOptions())

Lưu map thành file html độc lập

library(leaflet.extras)
# Save map thành file html
saveWidget(leaflet_object, file = "./map_transaction.html")

55.5 Cách lấy tọa độ chính xác từ Google API

Khi phân tích dữ liệu tọa độ địa lý ở thị trường Việt nam, một trong các điểm khó nhất là lấy được chính xác kinh độ và vĩ độ của khách hàng từ địa chỉ thực tế. Để lấy được tọa độ địa lý, ta có thể sử dụng package ggmap và đăng ký key API .

  • Bước 1: Tạo Google Project từ website console.could.google.com
  • Bước 2: Enable API google geocoding
  • Bước 3: Tạo Credential trong project và lấy key
  • Bước 4: Sử dụng ggmap để gọi API từ Google
library(dplyr)
library(ggmap)

######API theo email hoangducanh89
register_google(
  key = "xxx",
  account_type = "standard",
  day_limit = 100000
)

# Kiểm tra quotas
geocodeQueryCheck()
getOption("ggmap")

geocode("89 Láng Hạ", "latlona")
df <- data.frame(
  # customer id
  recid = c(1,2),
  # customer address
  address = c("89 Láng Hạ, Hà Nội", "20 Nguyen Chi Thanh, Dong Da, Hanoi"))
df
#Đặt progress bar
customer.address <- data.frame()
for (i in 1:nrow(df)) {
  print(paste0(i, " customers - ", round(i*100/nrow(df),2), "%"))
  #Kiểm tra query còn lại
  tryCatch({
    address <- geocode(df$address[[i]], "latlona", override_limit = T)
    address$id <- df$recid[[i]]
    address <- address %>% select(4, 1:3)
    customer.address <- rbind(customer.address, address)
  }, error = function(e) {
    print("error")
    cat("\n")
  },
  # Đặt warning để loại các trường hợp lỗi API trả về từ Google 
  warning = function(w) {
    print("warning")
    cat("\n")
  })
}
§

55.6 Tính toán khoảng cách giữa hai điểm

Khi phân tích về việc tối ưu hóa cho logistic, ta thường xuyên phải tính toán khoảng cách giữa hai hoặc nhiều điểm. Để xử lý vấn đề trên, ta có thể dùng package geosphere.

Cách tính khoảng cách dựa vào tọa độ của hai điểm như sau.
