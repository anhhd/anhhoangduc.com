17  Cách thức xử lý dữ liệu lớn

17.1 Giới thiệu

Khi phân tích dữ liệu, sẽ có một số trường hợp khối lượng dữ liệu, kể cả sau khi chiết xuất & rút gọn từ database, vẫn ở quy mô lớn đến hàng chục hoặc cả trăm triệu dòng đòi hỏi phân tích, tổng hợp. Với nhóm dữ liệu quy mô lớn này, có 1 số phương thức có thể sử dụng để phân tích như sau:

  • Cách 1: Tạo máy tính trên Cloud với RAM đủ lớn
  • Cách 2: Lưu trữ file & sử dụng các thư viện cho phép tính toán với file trên ổ cứng - như arrow.
  • Cách 3: Sử dụng một database tốc độ cao để có thể lưu trữ file và tổng hợp dữ liệu trực tiếp trên database đó - như duckdb
§

Ở chương này, ta tập trung vào phương án xử lý dữ liệu lớn với duckdb & arrow. Để sử dụng ví dụ với dữ liệu đủ lớn, ta dùng thư viện pacman::p_load(nycflights13)

17.2 Arrow

Với arrow, ta có thể đọc và xử lý dữ liệu từ file - hỗ trợ hai định dạng .csv và .parquet (hoặc .pq), mà không cần phải load dữ liệu vào RAM.

Ta có thể lưu dữ

# Load library
pacman::p_load(arrow, dplyr, nycflights13)
# Tạo file .parquet trên ổ cứng
nycflights13::flights %>% write_parquet("data/flights.parquet")

Xử lý dữ liệu với arrow, có 2 cách:

  • Đọc dữ liệu vào RAM với read_parquet - không nên dùng cách này
  • Tạo kết nối với file parquet với open_dataset - nên sử dụng
df_arrow <- open_dataset("data/flights.parquet")
# Kiểm tra dữ liệu
class(df_arrow)
[1] "FileSystemDataset" "Dataset"           "ArrowObject"      
[4] "R6"

Sau khi mở kết nối với dataset, arrow cho phép làm việc trực tiếp với file, hỗ trợ phần lớn các hàm với dplyr. Sau khi tính toán, có thể biến đổi thành dạng dataframe với hàm collect.

  • Ví dụ biến đổi dữ liệu
df_arrow %>% 
  mutate(new = dep_time * 10,
         group = case_when(
           dep_delay >= 15 ~ "3.very late",
           dep_delay >0 ~ "2.late",
           TRUE ~ "1.on time"
         )) %>% 
  select(dep_delay, group) %>% 
  head() %>% 
  collect()

17.3 DuckDB

duckdb là một database mới được phát triển gần đây và có tốc độ truy vấn và tổng hợp dữ liệu rất nhanh chóng. Tương tự như các database phổ biến khác, duckdb rất nhẹ và tinh gọn, cho phép lưu trữ dữ liệu với dạng file trên disk với định dạng .duckdb. Cách thức sử dụng và làm việc với ducdb có thể triển khai theo 2 cách

  • Cách 1: Sử dụng dbplyr, tạo kết nối với duckdb và làm việc như với 1 database bình thường.
  • Cách 2: Sử dụng duckplyr, cho phép sử dụng các hàm của dplyr với duckdb
pacman::p_load(duckdb, duckplyr, here)
con <- dbConnect(duckdb(), path = here("data", "my-db.duckdb"))
# Ghi dữ liệu và duckdb
dbWriteTable(con, "flights", flights)

Cách 2: Sử dụng duckplyr

Khác với cách thức sử dụng DBI, duckplyr cho phép biến đổi dữ liệu thành duckdb nhanh chóng. Với các hàm không hỗ trợ từ duckdb, sẽ tự động được chuyển thành dataframe để tính toán.

flight_duck <- flights %>% 
  as_duckplyr_df()
flight_duck %>% class
[1] "duckplyr_df" "tbl_df"      "tbl"         "data.frame" 
flight_duck %>% 
  group_by(year, month) %>% 
  summarise(
    mean = mean(dep_delay, na.rm = T),
    mean_trim = mean(dep_delay, trim = 0.01, na.rm = T)
  )

17.4 Kết luận

Như vậy, với khối lượng dữ liệu lớn, ta nên sử dụng phương thức sau để tính toán & xử lý dữ liệu.

  • Lưu trữ dữ liệu dưới dạng file parquet
  • Sử dụng arrow và duckdb đi kèm với DBI để xử lý và tổng hợp dữ liệu
